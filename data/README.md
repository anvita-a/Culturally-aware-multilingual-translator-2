# data/ — What Goes Here and How to Get It

This directory holds all datasets used by the pipeline.
Most files are generated by the scripts in this folder.
Some (GYAFC, X-FORMAL) must be downloaded manually.

---

## Step 1 — Build OPUS index (Week 1, run once, ~15 mins)

```bash
python data/build_opus_index.py
```

This downloads sentence pairs from OPUS-100 via HuggingFace and builds
the BM25 retrieval index. No account needed.

---

## Step 2 — Generate formality labels (Week 2, run once, ~20 mins)

```bash
# Generate 500 labels per language for all 7 languages without supervised data
python data/generate_formality_labels.py

# Then generate the spot-check sample (for κ computation)
python data/generate_formality_labels.py --spotcheck
```

After generating, send `data/spotcheck_sample.jsonl` to your spot-checkers
(native speakers / Prolific annotators). They fill in `verified_label` for
each entry. Then:

```bash
python data/generate_formality_labels.py --compute-kappa
```

This prints Cohen's κ — the number that goes in Table 1 of the paper.

---

## Step 3 — GYAFC (manual download required)

GYAFC requires a short request form to the authors (takes <24 hours):
https://github.com/raosudha89/GYAFC-corpus

Once approved, download and unzip into:
```
data/gyafc/
  Entertainment_Music/
    train/formal
    train/informal
    test/formal
    test/informal
  Family_Relationships/
    train/formal
    ...
```

**If you can't get GYAFC in time:** `train_formality.py` has a synthetic
fallback dataset that lets you test the full training pipeline. The paper
should use the real GYAFC for reported results.

---

## Step 4 — X-FORMAL (manual download)

GitHub: https://github.com/Elbria/xformal-FoST

Download the `en-fr` and `en-pt-br` splits and place them at:
```
data/xformal/
  fr/
    train.formal
    train.informal
    test.formal
    test.informal
  pt-br/
    train.formal
    ...
```

**If unavailable:** The training script works without X-FORMAL.
You'll have fewer supervised examples for French and Portuguese,
which will be slightly reflected in per-language formality accuracy.
Report this in the paper's limitations section.

---

## Quick start (if you just want to test the pipeline today)

You don't need any of the above to run `streamlit run app.py`.

The pipeline uses:
- A heuristic formality classifier if no trained model exists
- Empty few-shot examples if no OPUS index exists

Both fallbacks are documented in the code. Build the real data
before running final evaluations for the paper.
